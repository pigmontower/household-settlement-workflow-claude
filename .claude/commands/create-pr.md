**目的**: プルリクエストの作成

---

## 引数（オプション）

$ARGUMENTS: 紐付けるIssue番号（例: `issue 8` または `8`）

---

## Issue紐付けのルール

1. **引数でIssue番号が指定された場合**
   - `gh issue view <番号>` でIssueのタイトルを取得
   - 「【Issue #番号: タイトル】こちらと紐づけますか？紐付ける場合、以下どちらの方法にしますか？」と表示し、紐付け方法をユーザーに確認する:
     - `Closes #番号` : PRマージ時にIssueを自動クローズ
     - `Related to #番号` : 紐付けのみ（自動クローズしない）
   - ユーザーの選択に応じてPR本文に追加

2. **引数がない場合**
   - 会話のcontextから、現在取り組んでいるIssue情報（番号・タイトル）を探す
   - Issue情報が見つかった場合:
     - 「【Issue #番号: タイトル】こちらと紐づけますか？紐付ける場合、以下どちらの方法にしますか？」とユーザーに確認
     - ユーザーが紐付けを希望した場合、上記と同様に紐付け方法を確認
   - Issue情報が見つからない場合: 紐付けは行わない

## プルリクエスト作成のルール
- 本リポジトリに含まれる全てのGit差分は、プルリクエスト作成対象とする。コミットしてPR作成対応まで行うこと。
- featureブランチにチェックアウトしているか確認すること。featureブランチが作成されていなければ{featureブランチ作成コマンド}で作成する
- コミットメッセージは日本語で簡潔に作成すること
- プルリクエスト本文はフォーマットに従うこと
- プルリクエストを作成したら、`gh pr view --web`でブラウザで開くこと
- pushやPR作成のたびに、PRのURLをクリック可能な形式で表示すること
- `origin/main`へ直接pushすることは**いかなる状況でも厳禁**

## プルリクエスト本文のフォーマット
- 必ず `.github/PULL_REQUEST_TEMPLATE.md`を参照してフォーマットに従った形式で作成してください。
- なお、PRテンプレートに含まれるコメントアウトは、削除したうえで情報を埋めてください。
- 情報不足で埋められない場合は、'- 'など対象箇所で空文字を記載してください。

## featureブランチ作成コマンド
```bash
git checkout -b feature/$(date '+%Y%m%d%H%M%S') origin/main
```
