**目的**: コードレビューの実施

---

## レビュー対象

直近の変更（ステージングされた差分、または最新コミットの差分）をレビューしてください。

## 実行手順

1. `git diff --cached` または `git diff HEAD~1` で変更内容を確認
2. `code-reviewer` サブエージェントを使用してレビューを実施
3. レビュー結果を Critical / Warning / Suggestion の3段階で報告
4. レビュー完了後、「このレビュー結果をPRにコメントとして投稿しますか？」とユーザーに確認
5. ユーザーが投稿を希望した場合:
   - `gh pr list --head $(git branch --show-current)` で現在のブランチに関連するPRを検出
   - PRが見つかった場合、`gh pr comment` でレビュー結果を投稿
   - PRが見つからない場合、その旨を伝える

## 注意事項

- レビューは客観的な視点で行うため、サブエージェントを使用すること
- 問題がない場合も、その旨を報告すること
- PRへの投稿時は、CLAUDE.mdの署名ルールに従うこと
