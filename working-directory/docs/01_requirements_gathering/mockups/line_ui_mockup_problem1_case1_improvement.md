# line_ui_mockup_problem1_case1.html 改善提案書

## 問題点1: 現在処理中の品目がわかりにくい

### 元ファイルの該当箇所

```html
<div class="analysis-card">
    <span class="highlight">【品目A: 500円】</span><br>
    分類を選択してください。<br>
    <small style="color: #888;">AI推論：割り勘（食材のため）</small>
</div>
```

```html
<div class="status-msg">
    残り 3品目あります
</div>
```

### なにが問題か

1. 「残り3品目」と表示されているが、全部で何品目中なのかがわからない
2. 品目が1つずつ表示されるため、レシート全体の中でどこを処理しているかが把握しにくい
3. 案4で追加された「進捗可視化」機能が反映されていない

### 改善内容

案4の進捗可視化UIを採用し、全品目を一覧表示しながら現在処理中の品目を絵文字でハイライトするUIに変更。

---

## 問題点2: 「全て割り勘」ボタンの位置と動作が不明確

### 元ファイルの該当箇所

```html
<button class="qr-button" style="border-color: #666; color: #666;">⏩ 全て割り勘</button>
```

### なにが問題か

1. 「全て割り勘」ボタンがグレーアウトされた見た目になっており、押せるかどうかわかりにくい
2. 案5で追加された「キャンセル」ボタンがない

### 改善内容

1. 「全て割り勘」ボタンを明確なスタイルに変更
2. 「キャンセル」ボタンを追加

---

## 問題点3: 案5のキャンセル機能が反映されていない

### 元ファイルの該当箇所

（キャンセルボタンが存在しない）

### なにが問題か

案5で追加された「❌ キャンセル」ボタンがないため、ユーザーが処理を中断できない。

### 改善内容

クイックリプライに「❌ キャンセル」ボタンを追加。

---

## 改善後のモック

改善後のUIは `line_ui_mockup_problem1_case1_improvement.html` を参照してください。

主な変更点：
1. 全品目を進捗絵文字付きで一覧表示（案4の機能）
2. 「❌ キャンセル」ボタンを追加（案5の機能）
3. 「⏩ 全て割り勘」ボタンのスタイルを改善
