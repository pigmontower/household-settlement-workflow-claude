# line_ui_mockup_problem1_case3.html 改善提案書

---

## 問題点1: 「登録」ボタンがない

### 元ファイルの該当箇所

```html
<div class="quick-reply-container">
    <button class="qr-button">✏️ 訂正</button>
    <button class="qr-button primary">📂 仕分けへ</button>
</div>
```

### なにが問題か

要件定義書（REQUIREMENTS.md）では「解析結果表示後のボタン」として `[訂正] [仕分け] [登録]` の3つが定義されていますが、モックには「登録」ボタンがありません。

「登録」ボタンは「全品目を割り勘として一括登録」する簡易モードとして必要な機能です。

### 改善内容

クイックリプライに「📋 登録（全て割り勘）」ボタンを追加。

---

## 問題点2: 「キャンセル」ボタンがない

### 元ファイルの該当箇所

（キャンセルボタンが存在しない）

### なにが問題か

案5で追加された「❌ キャンセル」ボタンがないため、ユーザーが間違った画像を送った場合などに処理を中断できない。

### 改善内容

クイックリプライに「❌ キャンセル」ボタンを追加。

---

## 問題点3: OCR誤認識の例がわかりにくい

### 元ファイルの該当箇所

```html
品目B：<span class="correction-marker">3D0</span>
```

### なにが問題か

OCRの誤認識例として「3D0」（300の誤認識）が示されていますが、ユーザーにとって何が間違いなのかわかりにくい。

### 改善内容

誤認識箇所に取り消し線を追加し、より視覚的に問題箇所を強調。
