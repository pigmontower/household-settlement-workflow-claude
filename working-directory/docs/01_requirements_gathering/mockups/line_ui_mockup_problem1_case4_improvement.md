## 問題点1: 「キャンセル」ボタンがない

### 元ファイルの該当箇所

```html
<div class="quick-reply-container">
    <button class="qr-button primary">🤝 割り勘</button>
    <button class="qr-button">🙋‍♂️ 自分</button>
    <button class="qr-button">🙋‍♀️ 妻</button>
    <button class="qr-button">⏳ 保留</button>
</div>
```

### なにが問題か

案5で追加された「❌ キャンセル」ボタンがありません。仕分け中に処理を中断したい場合に対応できません。

### 改善内容

クイックリプライに「❌ キャンセル」ボタンを追加。

---

## 問題点2: 「保留」の扱いが不明確

### 元ファイルの該当箇所

```html
<button class="qr-button">⏳ 保留</button>
```

### なにが問題か

「保留」を選択した場合の挙動が不明確です：
- 保留した品目はどうなるのか
- 保留した品目に後から戻れるのか
- スプレッドシートにはどう記録されるのか

### 改善内容

「保留」ボタンを削除し、代わりに以下の運用とする：
- 一時的にスキップしたい場合は「キャンセル」で中断し、後でセッション再開
- または全品目を処理してから、スプレッドシート上で手動修正

---

## 問題点3: 進捗表示にコメントテキストがある

### 元ファイルの該当箇所

```html
➡️ 品目A：500（現在処理中）
```

### なにが問題か

進捗表示に「（現在処理中）」というコメントが含まれていますが、実際のLINEメッセージでは絵文字だけで十分意味が伝わります。テキストが増えると視認性が下がります。

### 改善内容

進捗表示からコメントテキストを削除し、絵文字のみでシンプルに表示。
