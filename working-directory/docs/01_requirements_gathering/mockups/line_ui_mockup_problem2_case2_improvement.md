## 問題点1: 表示されている金額が「妻負担額」かどうか不明

### 元ファイルの該当箇所

```html
<div class="list-content">【清算額内訳】

    2024/12/20 スーパーABC
    → 1,500円

    2024/12/22 ドラッグストアXYZ
    → 800円

    2024/12/25 コンビニ123
    → 350円

    ────────────
    <span class="total-highlight">合計：2,650円</span>
</div>
```

### なにが問題か

表示されている金額（1,500円、800円など）が「購入金額」なのか「妻負担額」なのかが不明です。

本システムの目的は「妻が支払う分の料金を計算して妻に伝える」ことなので、表示されるべきは「妻負担額」ですが、現状のUIではその区別がつきません。

### 改善内容

未清算リスト表示を以下のように改善：
- 各明細に「妻負担額」であることを明示
- 合計欄に「妻負担合計」と明記

## 問題点2: 品目レベルの詳細が見えない

### 元ファイルの該当箇所

（上記と同じ）

### なにが問題か

店舗ごとの合計金額しか表示されていないため、どの品目がどのように仕分けされたかが確認できません。

ただし、LINEのメッセージで品目詳細まで表示すると長くなりすぎる問題もあります。

### 改善内容

リッチメニューに「スプレッドシートを開く」ボタンを追加し、詳細確認はスプレッドシートで行う導線を設ける。

---

## 問題点3: 清算確定後の動作説明がない

### 元ファイルの該当箇所

（清算確定ボタン押下後の画面がない）

### なにが問題か

「清算済みにする」ボタンを押した後にどうなるかがUIで示されていません。

### 改善内容

清算確定後のメッセージ例を追加：
- 清算が確定したこと
- 新しいシートが作成されたこと
- スプレッドシートへのリンク
